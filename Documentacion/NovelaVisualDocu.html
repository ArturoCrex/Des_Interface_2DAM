Parte 1 Configuración y Assets Generados

1.1 Estructura del Proyecto (Monogatari)

Configuración inicial del sistema de archivos para el motor de la novela visual. Usamos Monogatari, un framework de Java Script que funciona como motor para hacer juegos en web, especializado en novelas visuales. Decidi usar esto como motor para facilitar configuraciones de fondos, mecanicas de juego y hacer que funcione màs fluido el juego.

~/project> mkdir assets/images/scenes
~/project> mkdir assets/images/items
~/project> mkdir js/minigames
~/project> touch js/storage.js

Parte 2 Historial de Prompts (Desarrollo)

Evolución paso a paso del proyecto, desde la estructura base hasta la integración de minijuegos y backend.

Fase 1: Petición Inicial y Estructura

Configuración del entorno Monogatari y definición de variables de inventario.

"Vamos a crear una juego novela visual... quiero mantenerlo sencillo usando html, css y java script, usando el framework de monogatari.
tiene que existir un archivo storage.js... inventory: { bisturi: false, tarjeta: false, celula: false, reactor_fix: false }..."
Acciones realizadas: Descarga del framework, limpieza de archivos core, creación de carpetas de assets y configuración de storage.js.

Fase 2: Implementación de la Historia

Integración del guion completo .

Ahora añadiras el script de la historia que te coy a dar (Guion completo de la historia que quieras), ademas asegurate de crear las variables necesarias para los siguietes objetos, fondos y personajes(Insertar lista de objetos con descripcion de que es cada uno).
Acciones realizadas: Programación de escenas (Enfermería, Pasillo, Lab, Ingeniería, Puente), lógica de recolección de ítems y programación de los 3 finales distintos + muerte.

Fase 3: Refinamiento de UI y Assets

Mejora de la legibilidad y limpieza de assets no utilizados.

Debido a un cambio de planes elimina el marco de la UI originalmente dado como foto. Ademas actualiza el css para darle un estilo cercano a como se veria un testo en otro juegos, sin que moleste en ver las imagenes de fondo
Acciones realizadas: Eliminación de ui_terminal, actualización de main.css para fondos semitransparentes en cajas de texto y validación de directorios.

Fase 4: Refactorizacion del juego y solucionar bugs

Creación de una logica lineal para que le juego funcione.

Crea la logica para que el juego sea lineal, añadiendo modificaciones necesarias a la historia para que sea visible, entendible y lo mas importante de todo, que sea rapida y funcional.
Acciones realizadas:
  • Creación de interacciones entre partes del juego.
  • Creación de elecciones para incluir al usuario.
  • Integración de Promesas (opciones y validaciones) para manejar estados de Victoria/Derrota.

Fase 5: Integración con Supabase (Deprecated)

Seguir la elecciones de jugadores y mostrarlas a tiempo real a traves de supabase

Añade lo necesario al codigo para que la base de datos que te doy pueda seguir las estadisticas de eleccion del usuario, ademas muestralas con un recuadro el porcentaje de elección
Acciones realizadas: Decidi eliminar esta parte, debido a que causaba errores de desincronización si se ejecutaba en dos sitios al mismo tiempo y podia dar lag y tardar en cargas bastante para un proyecto tan pequeño.

Fase 6: Pulido Final

Actualizar pantalla de título y actualización de documentación.

Crea una pagina principal actualizada con las opciones base de Monogatari y usa de fondo la iamgen que he añadido como nueva.
Acciones realizadas: Estilizado del Menú Principal de Monogatari.

Declarar monogatari en el codigo

Declarando los assets para monogatari

Monogatari se le tiene que dar una lista de assets que entienda que y porque se hacen cosas. Asi declaramos objetos, fondos y personajes.

Crear personajes en monogatari

En el siguiente codigo podemos ver como se crean personajes en monogatari, dandole sprites, id unico y permitiendonos usarlo de manera facil e intuitiva.

Condiciones para elegir

Monogatari permite ejecutar partes del juego y decisiones de manera facil con condicionales, permitiendo elegir la historia

Script de la historia

Monogatari permite iniciar el script y ejecutar la historia llamando a shortcuts cuando sea necesario, para ejecutar funciones y acciones por el usuario.

Crear inventario

Si tienes items puedes crear un inventario, que permite saber si el usuario tiene o no objetos y luego acceder a esa variable para determinar por que caminos va el juego.

Opciones

En monogatari es importante declarar donde se encuentra cada parte del contenido, fotos, personajes, audios. Si se declara mal el juego no funcionara correctamente.
