Documentación: Gestor de Documentos

Registro del desarrollo de una aplicación web para gestión, edición y conversión de archivos, usando React, Vite y Supabase.

Parte 1 Configuración del Entorno y Pasos Previos

1.1 Creación del proyecto con Vite + React

Si aún no tienes el proyecto, créalo con los comandos oficiales de Vite (plantilla React).

~> npm create vite@latest aGravity -- --template react
~> cd ruta-proyecto
~/ruta-proyecto> npm install

1.2 Instalación de dependencias

Instala las librerías necesarias para todas las funcionalidades de aGravity.

~/aGravity> npm install @supabase/supabase-js xlsx heic2any jszip file-saver lucide-react
# @supabase/supabase-js → cliente oficial
# xlsx → lectura/escritura Excel
# heic2any → conversión HEIC → JPEG
# jszip + file-saver → creación y descarga de ZIP
# lucide-react → iconos modernos

1.3 Configuración de Supabase (.env.local)

Crea un archivo .env.local en la raíz del proyecto con tus credenciales de Supabase.

VITE_SUPABASE_URL=https://tu-proyecto.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

1.4 Crear tabla en Supabase (SQL obligatorio)

Ve a tu proyecto de Supabase → SQL Editor → pega y ejecuta lo siguiente: 

CREATE TABLE documents (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users NOT NULL,
  name TEXT NOT NULL,
  content TEXT,               -- NULL para binarios (PDF, imágenes, etc.)
  storage_path TEXT,          -- ruta en bucket 'files' para binarios
  file_type TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Políticas RLS básicas
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Usuarios ven sus propios archivos" ON documents
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Usuarios insertan sus archivos" ON documents
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Usuarios actualizan sus archivos" ON documents
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Usuarios eliminan sus archivos" ON documents
  FOR DELETE USING (auth.uid() = user_id);

También crea un bucket llamado files en Storage (público o con RLS según prefieras).

1.5 Ejecutar el proyecto

~/aGravity> npm run dev
Local: http://localhost:5173
Network: http://192.168.x.x:5173

Abre el enlace y ya tendrás la aplicación funcionando completamente.

Parte 3 Galería de Código y Lógica Interna

Análisis profundo de la implementación técnica basado en la documentación "Explicación del Código de aGravity".

3.1. Arquitectura y Seguridad

Entrada de datos a la aplicación (App.jsx)

// Gestión de navegación por estado (SPA)
const [activeTab, setActiveTab] = useState('browser')

const renderContent = () => {
  if (activeTab === 'browser') return <FileBrowser />
  if (activeTab === 'tools') return <Converter />
  // Renderizado condicional sin recarga
}

Permite la entrada de datos a traves de los archivos del dispositivo y de las base de datos.

Protección de Rutas (Auth.jsx)

useEffect(() => {
  supabase.auth.getSession().then(({ data: { session } }) => {
    setSession(session)
  })
}, [])

if (!session) return <Auth />

Bloqueo de seguridad: Si no hay sesión de Supabase activa, renderiza el Login en lugar de la App.

3.2. Sistema de Archivos Inteligente

Validación de Entrada (FileUploader)

const handleFileLoaded = (content, name) => {
  const ext = name.split('.').pop().toLowerCase()
  
  // Lógica de filtrado según la pestaña activa
  if (activeTab === 'editor' && !['txt','md','json'].includes(ext)) {
     alert("Formato no soportado en editor")
  }
}

Detección de extensión al soltar el archivo para dirigirlo al procesador correcto.

Estrategia de Guardado Híbrido

// 1. Binarios (PDF/Img) -> Storage
if (isBinary) {
  await supabase.storage.from('files').upload(path, fileObj)
}

// 2. Metadatos y Texto -> Base de Datos
await supabase.from('documents').insert([{
   name: fileName,
   storage_path: isBinary ? path : null,
   content: !isBinary ? finalContent : null
}])

Sistema dual: El texto se guarda en SQL para búsqueda rápida; los binarios en Bucket S3, guardando solo la referencia.

Apertura Inteligente (App.jsx)

if (fileRec.content !== null) {
  setFileContent(fileRec.content) // Carga instantánea (Texto)
} else if (fileRec.storage_path) {
  // Descarga bajo demanda (Binario)
  const { data } = await supabase.storage
      .from('files')
      .download(fileRec.storage_path)
}

Optimización de carga: Solo descarga los archivos pesados cuando el usuario hace clic en ellos.

3.3. Motores de Conversión y Edición

Lector de Excel (xlsx)

import * as XLSX from 'xlsx'

const workbook = XLSX.read(e.target.result, { type: 'binary' });
const sheetName = workbook.SheetNames[0];
// Conversión automática a JSON
const json = XLSX.utils.sheet_to_json(workbook.Sheets[sheetName]);

Uso de SheetJS para parsear hojas de cálculo binarias y convertirlas a estructuras manipulables en web.

Conversor HEIC (heic2any)

import heic2any from 'heic2any'

const blob = await heic2any({
  blob: file,
  toType: 'image/jpeg',
  quality: 0.8
})
const url = URL.createObjectURL(blob)

Procesa imágenes de Apple directamente en el navegador, devolviendo un JPEG visible.

Empaquetador ZIP (JSZip)

import JSZip from 'jszip'
const zip = new JSZip()

zip.file("nota.txt", "contenido texto");
zip.file("imagen.png", blobData);

const content = await zip.generateAsync({ type: 'blob' })
saveAs(content, 'backup.zip')

Generación de archivos comprimidos en el cliente para exportar lotes de documentos.

Editor Hexadecimal

const buffer = e.target.result; // ArrayBuffer
const view = new Uint8Array(buffer);
// Renderiza cada byte en base 16
view.forEach(byte => {
  console.log(byte.toString(16).padStart(2, '0'))
})

Permite la inspección de bajo nivel para archivos corruptos o formatos desconocidos.
