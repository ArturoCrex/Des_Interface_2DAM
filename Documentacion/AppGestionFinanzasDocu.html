Documentación: App Gestión Finanzas

Registro del desarrollo de una aplicación PWA offline-first para gestión de finanzas personales con integración de múltiples APIs de mercado y sincronización en la nube.

Parte 1 Configuración del Entorno y Stack Tecnológico

1.1 Estructura del Proyecto

La aplicación es una PWA (Progressive Web App) construida con Vite + React 19 + TypeScript, diseñada para funcionar tanto online como offline.

~> npm create vite@latest AppGestionFinanzas -- --template react-ts
~> cd AppGestionFinanzas
~/AppGestionFinanzas> npm install

1.2 Instalación de Dependencias Principales

Stack completo con gestión de estado, base de datos offline, internacionalización y visualización de datos.

1.3 Resumen de APIs Utilizadas

La aplicación integra múltiples APIs para proporcionar datos de mercado en tiempo real y funcionalidad cloud.

¿Por qué estas APIs?

Supabase (Auth + Database + Storage)

Razón: Proporciona autenticación completa, base de datos PostgreSQL en la nube y almacenamiento de archivos en un solo servicio. Permite sincronización multi-dispositivo y backup automático. Su capa gratuita (50K usuarios activos mensuales) es generosa para proyectos personales.

Variables: VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY

Open Exchange Rates (Divisas)

Razón: API gratuita sin necesidad de registro que proporciona tasas de cambio actualizadas diariamente para más de 200 monedas. Sin límites de peticiones, ideal para consultas frecuentes sin costes.

Endpoint: https://open.er-api.com/v6/latest/{BASE} | API Key: No requerida

CoinGecko (Criptomonedas)

Razón: La API más completa y confiable para datos de criptomonedas. Proporciona precios en tiempo real, capitalización de mercado, cambios porcentuales y metadatos (logos, nombres) sin necesidad de API key. Límite de ~30 peticiones/minuto es suficiente para uso personal.

Endpoint: https://api.coingecko.com/api/v3/coins/markets | API Key: Opcional

Metales Preciosos (Cálculo Híbrido)

Razón: Implementación híbrida que usa precios base en USD (actualizados manualmente) y los convierte a otras monedas usando Open Exchange Rates. Evita depender de APIs de metales que suelen tener límites muy restrictivos en sus planes gratuitos.

Nota: Para precios en tiempo real, se recomienda integrar metals.dev o goldapi.io

Alpha Vantage (Acciones)

Razón: API gratuita con registro que proporciona cotizaciones de acciones en tiempo real, búsqueda de símbolos y datos históricos. Límite de 5 peticiones/minuto y 500/día es adecuado para consultas puntuales. Alternativa gratuita a APIs premium como Bloomberg o Reuters.

Variable: VITE_ALPHA_VANTAGE_KEY | Registro: alphavantage.co

Dexie.js (IndexedDB Local)

Razón: No es una API externa, pero es crucial para el modo offline-first. Dexie simplifica el uso de IndexedDB (base de datos del navegador) permitiendo almacenar transacciones, metas de ahorro y suscripciones localmente. La app funciona completamente sin conexión gracias a esta capa.

1.4 Configuración de Variables de Entorno

Crea un archivo .env en la raíz del proyecto con las siguientes variables:

# Alpha Vantage (opcional, para cotización de acciones)

Sin estas variables, la app funciona en modo offline usando únicamente IndexedDB.

Parte 2 Historial de Prompts y Evolución del Proyecto

Evolución paso a paso del proyecto, desde la concepción inicial hasta la implementación de características avanzadas.

Prompt 1: Definición de la Arquitectura Base

Establecimiento de la estructura fundamental de la aplicación con enfoque offline-first y PWA.

"Necesito crear una aplicación de gestión de finanzas personales que funcione como PWA. Debe tener capacidad offline usando IndexedDB, sincronización con Supabase cuando haya conexión, y soporte para múltiples idiomas (español e inglés). La estructura debe ser modular con React + TypeScript + Vite."
Objetivo:

Crear una base sólida que permita trabajar offline-first, garantizando que la aplicación funcione sin conexión a internet y sincronice datos cuando esté disponible. Implementar internacionalización desde el inicio para facilitar expansión futura.

Resultado:

Se estableció la arquitectura con Dexie.js para IndexedDB, Zustand para gestión de estado global, React Router v7 para navegación, y i18next para internacionalización. Se configuró vite-plugin-pwa para generar Service Workers automáticamente.

Prompt 2: Sistema de Autenticación Híbrido

Implementación de autenticación que funcione tanto en modo cloud como offline.

"Implementa un sistema de autenticación que use Supabase Auth cuando esté disponible, pero que permita usar la app en modo offline sin login. Debe detectar automáticamente si Supabase está configurado y adaptar el flujo de autenticación."
Objetivo:

Permitir que usuarios sin configuración de Supabase puedan usar la app inmediatamente en modo offline, mientras que usuarios con credenciales cloud obtengan sincronización multi-dispositivo y backup automático.

Resultado:

Se creó useAuthStore con detección automática de Supabase mediante isSupabaseEnabled(). El componente AuthGuard permite acceso directo en modo offline o redirige a login si Supabase está configurado. Los servicios de datos (dataService.ts) priorizan Supabase pero funcionan con Dexie como fallback.

Prompt 3: Integración de APIs de Mercado

Añadir visualización de datos de mercado financiero en tiempo real.

"Crea una sección de Mercados que muestre cotizaciones de divisas (forex), criptomonedas, metales preciosos y acciones. Usa APIs gratuitas sin necesidad de registro cuando sea posible. Implementa caché para evitar exceder límites de peticiones."
Objetivo:

Proporcionar información de mercado relevante para usuarios que gestionan inversiones o monedas extranjeras, sin depender de APIs de pago y respetando límites de rate limiting.

Resultado:

Se implementó marketService.ts con cuatro fuentes de datos: Open Exchange Rates (forex), CoinGecko (crypto), cálculo híbrido para metales preciosos, y Alpha Vantage (acciones). Se añadió caché en localStorage para reducir peticiones y mejorar rendimiento. La página Markets muestra tabs con visualización de datos usando Recharts.

Prompt 4: Sistema de Adjuntos y Almacenamiento

Permitir adjuntar recibos, facturas y documentos a las transacciones.

"Implementa la capacidad de adjuntar archivos (imágenes, PDFs, audio) a las transacciones. En modo offline, guarda los archivos como base64 en IndexedDB. En modo cloud, súbelos a Supabase Storage y guarda solo la referencia."
Objetivo:

Permitir documentación completa de gastos e ingresos mediante adjuntos, manteniendo la funcionalidad offline-first y optimizando el almacenamiento en cloud.

Resultado:

Se creó fileService.ts que maneja archivos de forma híbrida: en modo offline convierte archivos a base64 y los almacena en la tabla attachments de Dexie; en modo cloud sube a Supabase Storage y guarda solo la URL. Se implementó previsualización de imágenes y descarga de PDFs directamente desde la interfaz de transacciones.

Prompt 5: Metas de Ahorro y Contribuciones

Sistema de seguimiento de objetivos de ahorro con visualización de progreso.

"Añade una sección de Ahorros donde los usuarios puedan crear metas (ej: 'Vacaciones - €2000'), registrar contribuciones periódicas, y ver el progreso con barras visuales y porcentajes. Debe calcular cuánto falta y la fecha estimada de cumplimiento."
Objetivo:

Motivar el ahorro mediante visualización clara del progreso hacia objetivos específicos, con cálculos automáticos de proyecciones basadas en contribuciones históricas.

Resultado:

Se implementaron las tablas saving_goals y saving_contributions tanto en Dexie como en Supabase. La página Savings muestra tarjetas con barras de progreso, porcentaje completado, y estimación de fecha de cumplimiento basada en la media de contribuciones mensuales. Se añadió modal para registrar nuevas contribuciones con validación de montos.

Prompt 6: División de Gastos (Split Bills)

Funcionalidad para dividir gastos entre múltiples personas con cálculo automático de deudas.

"Implementa una calculadora de división de gastos donde pueda añadir participantes, asignar quién pagó qué, y calcular automáticamente quién debe dinero a quién. Debe optimizar las transacciones para minimizar el número de pagos necesarios."
Objetivo:

Facilitar la gestión de gastos compartidos (cenas, viajes, alquileres) con cálculo automático de deudas y optimización de transacciones para simplificar liquidaciones.

Resultado:

Se creó la página SplitBill con algoritmo de optimización de deudas que reduce el número de transacciones necesarias. Permite añadir participantes dinámicamente, registrar gastos individuales, y genera un resumen visual de quién debe a quién. Los registros se guardan en split_bills con historial completo de liquidaciones.

Prompt 7: Despliegue en GitHub Pages

Configuración para deployment estático en GitHub Pages con manejo correcto de rutas.

"Configura el proyecto para desplegarse en GitHub Pages. Asegúrate de que las rutas funcionen correctamente con HashRouter, que los assets se carguen desde la ruta base correcta, y que las variables de entorno se manejen adecuadamente en el build."
Objetivo:

Hacer la aplicación accesible públicamente sin necesidad de servidor backend, aprovechando GitHub Pages como hosting gratuito para sitios estáticos.

Resultado:

Se configuró vite.config.ts con base: '/AppGestionFinanzas/' para rutas correctas. Se cambió de BrowserRouter a HashRouter para evitar problemas con rutas en GitHub Pages. 3.1. Arquitectura de Routing y Lazy Loading

App.tsx - Configuración de Rutas con Code Splitting 

// Lazy-loaded pages para optimizar bundle size
const Dashboard = lazy(() => import('@/pages/Dashboard/Dashboard'));
const Transactions = lazy(() => import('@/pages/Transactions/Transactions'));
const Savings = lazy(() => import('@/pages/Savings/Savings'));
const Markets = lazy(() => import('@/pages/Markets/Markets'));

const router = createHashRouter([
  // Public routes
  { path: '/login', element: <Login /> },
  { path: '/register', element: <Register /> },

  // Protected routes
  {
    path: '/',
    element: (
      <AuthGuard>
        <AppShell />
      </AuthGuard>
    ),
    children: [
      { index: true, element: <Dashboard /> },
      { path: 'transactions', element: <Transactions /> },
      { path: 'savings', element: <Savings /> },
      { path: 'markets', element: <Markets /> },
    ],
  },
]);

Decisión clave: HashRouter en lugar de BrowserRouter para compatibilidad con GitHub Pages. Lazy loading reduce el bundle inicial de ~800KB a ~200KB, cargando páginas bajo demanda.

3.2. Base de Datos Offline (Dexie + IndexedDB)

db.ts - Esquema de Base de Datos Local

import Dexie, { type Table } from 'dexie';

class FinanceDB extends Dexie {
    transactions!: Table<Transaction>;
    attachments!: Table<Attachment>;
    savingGoals!: Table<SavingGoal>;
    savingContributions!: Table<SavingContribution>;
    subscriptions!: Table<Subscription>;
    splitBills!: Table<SplitBillRecord>;

    constructor() {
        super('FinanzasApp');

        // Versión 3 con índices optimizados
        this.version(3).stores({
            transactions: '++id, type, category, date, createdAt',
            attachments: '++id, transactionId, mimeType',
            savingGoals: '++id, name, deadline',
            savingContributions: '++id, goalId, date',
            subscriptions: '++id, name, nextPayment, active',
            splitBills: '++id, title, createdAt',
        });
    }
}

export const db = new FinanceDB();

Ventaja: IndexedDB permite almacenar datos estructurados sin límite de tamaño (a diferencia de localStorage que tiene ~5MB). Dexie simplifica las queries con sintaxis tipo SQL: db.transactions.where('type').equals('expense').toArray()

3.3. Gestión de Estado Global (Zustand)

useAuthStore.ts - Estado de Autenticación

import { create } from 'zustand';

interface AuthState {
    user: User | null;
    session: Session | null;
    isAuthenticated: boolean;
    isOfflineMode: boolean;
    
    initialize: () => Promise<void>;
    logout: () => Promise<void>;
}

export const useAuthStore = create<AuthState>()((set) => ({
    user: null,
    session: null,
    isAuthenticated: false,
    isOfflineMode: !isSupabaseEnabled(),

    initialize: async () => {
        // Si Supabase no está configurado, modo offline automático
        if (!isSupabaseEnabled()) {
            set({ isOfflineMode: true, isAuthenticated: true });
            return;
        }

        // Si está configurado, verificar sesión
        const { session } = await getSession();
        set({
            user: session?.user || null,
            session,
            isAuthenticated: !!session,
        });

        // Escuchar cambios de autenticación
        onAuthStateChange((user, session) => {
            set({ user, session, isAuthenticated: !!session });
        });
    },
});

Patrón offline-first: La app asume modo offline por defecto y habilita cloud solo si detecta credenciales de Supabase. Esto garantiza que funcione inmediatamente sin configuración.

3.4. Servicio de Mercados (APIs Externas)

Forex - Open Exchange Rates

const FOREX_BASE_URL = 'https://open.er-api.com/v6/latest';

export async function getForexRates(base = 'EUR') {
    const res = await fetch(`${FOREX_BASE_URL}/${base}`);
    const data = await res.json();
    
    return {
        base: data.base_code,
        rates: data.rates,
        updatedAt: new Date(data.time_last_update_utc),
    };
}

Devuelve tasas de cambio de la moneda base a ~200 monedas. Sin API key, sin límites.

Crypto - CoinGecko

export async function getCryptoPrices(
    coins = ['bitcoin', 'ethereum', 'solana'],
    currency = 'eur'
) {
    const ids = coins.join(',');
    const res = await fetch(
        `${COINGECKO_URL}/coins/markets?vs_currency=${currency}&ids=${ids}`
    );
    const data = await res.json();

    return data.map((coin) => ({
        id: coin.id,
        symbol: coin.symbol.toUpperCase(),
        price: coin.current_price,
        change24h: coin.price_change_percentage_24h,
        marketCap: coin.market_cap,
        image: coin.image,
    }));
}

Proporciona precio, cambio 24h, capitalización de mercado y logo. Límite: ~30 req/min sin key.

Metales - Cálculo Híbrido

export async function getMetalPrices(currency = 'EUR') {
    // Obtener tasa de conversión USD -> currency
    const rates = await getForexRates('USD');
    const conversionRate = rates.rates[currency] || 1;

    // Precios base en USD/oz (actualizados manualmente)
    const goldUSD = 5057;
    const silverUSD = 78;
    const platinumUSD = 2077;

    return [
        {
            name: 'Oro',
            symbol: 'XAU',
            pricePerOz: goldUSD * conversionRate,
            currency,
        },
        // ... plata, platino
    ];
}

Usa precios base USD + conversión forex. Para tiempo real, integrar metals.dev.

Acciones - Alpha Vantage

const ALPHA_VANTAGE_KEY = import.meta.env.VITE_ALPHA_VANTAGE_KEY || 'demo';

export async function getStockQuote(symbol: string) {
    const res = await fetch(
        `https://www.alphavantage.co/query?function=GLOBAL_QUOTE&symbol=${symbol}&apikey=${ALPHA_VANTAGE_KEY}`
    );
    const data = await res.json();
    const quote = data['Global Quote'];

    return {
        symbol: quote['01. symbol'],

        change: parseFloat(quote['09. change']),
        changePercent: parseFloat(quote['10. change percent']),
        currency: 'USD',
    };
}

Requiere API key gratuita. Límite: 5 req/min, 500 req/día.

3.5. Sincronización Híbrida (Supabase + Dexie)

dataService.ts - Patrón de Fallback

export async function getTransactions() {
    // Prioridad 1: Supabase (si está configurado y hay sesión)
    if (isSupabaseEnabled() && supabase) {
        const { data, error } = await supabase
            .from('transactions')
            .select('*')
            .order('date', { ascending: false });
        
        if (!error) return data;
    }

    // Fallback: Dexie (modo offline)
    return await db.transactions
        .orderBy('date')
        .reverse()
        .toArray();
}

export async function createTransaction(transaction: Transaction) {
    // Intentar guardar en Supabase primero
    if (isSupabaseEnabled() && supabase) {
        const { data, error } = await supabase
            .from('transactions')
            .insert(transaction)
            .select()
            .single();
        
        if (!error) {
            // Sincronizar a Dexie como caché local
            await db.transactions.add(data);
            return data;
        }
    }

    // Si falla o no hay Supabase, guardar solo en Dexie
    const id = await db.transactions.add(transaction);
    return { ...transaction, id };
}

Estrategia: Intentar cloud primero, fallback a local. Mantener caché en Dexie incluso con Supabase activo para acceso rápido.

3.6. Internacionalización (i18next)

i18n.ts - Configuración de Idiomas

import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import Backend from 'i18next-http-backend';

i18n
    .use(Backend) // Carga traducciones desde /public/locales/{lng}/translation.json
    .use(LanguageDetector) // Detecta idioma del navegador
    .use(initReactI18next)
    .init({
        fallbackLng: 'es',
        supportedLngs: ['es', 'en'],
        interpolation: {
            escapeValue: false, // React ya escapa por defecto
        },
    });

export default i18n;

Uso en componentes: const { t } = useTranslation(); return <h1>{t('dashboard.title')}</h1>
