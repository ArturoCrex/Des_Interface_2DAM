Documentación: Keep Clon

Registro detallado del desarrollo, comandos de consola para la configuración del entorno React + Vite, y evolución de los prompts del proyecto de clonar una aplicación para clase.

Parte 1 Configuración del Entorno (Usando la CMD)

1. Creación del Proyecto con Vite

Verificamos la versión de Node.js (v24.11.1) y creamos la estructura base del proyecto usando la plantilla de React.
Nota Importante: Se debe añadir --template react a todos los comandos de instalacion relacionados a la version, este comando basicamente hace que podamos instalar todo para la version mas nueva (v24.11.1).

C:\Users\Arturo> node -v
v24.11.1

C:\Users\Arturo> cd Desktop

C:\Users\Arturo\Desktop> npm create vite@latest keep-clon -- --template react
Need to install the following packages:
create-vite@8.2.0
Ok to proceed? (y) y

> npx
> create-vite keep-clon --template react

Scaffolding project in C:\Users\Arturo\Desktop\keep-clon...
Done. Now run:

  cd keep-clon
  npm install
  npm run dev

2. Instalación de Dependencias y Tailwind CSS

Entramos al directorio, instalamos las dependencias base (`npm install`). A continuación, instalamos Tailwind CSS.
Nota Importante: Se detectaron problemas de compatibilidad con la versión automática de Tailwind, por lo que se forzó la instalación de la versión 3.4.17 para asegurar estabilidad.

C:\Users\Arturo\Desktop> cd keep-clon

C:\Users\Arturo\Desktop\keep-clon> npm install
added 157 packages, and audited 158 packages in 10s

C:\Users\Arturo\Desktop\keep-clon> npm install -D tailwindcss@3.4.17 postcss autoprefixer --legacy-peer-deps
added 58 packages, changed 1 package, and audited 244 packages in 4s
found 0 vulnerabilities

3. Supabase, Iconos y Canvas

Instalación de librerías para la base de datos (Supabase), iconos y dibujo.
Solución de error: Debido a un conflicto de dependencias con react-canvas-draw y React 19, se utilizó el flag --legacy-peer-deps para permitir la instalación. Finalmente, abrimos el editor de código.

C:\Users\Arturo\Desktop\keep-clon> npm install @supabase/supabase-js react-icons react-masonry-css react-canvas-draw --legacy-peer-deps
added 23 packages, and audited 186 packages in 3s

Prompt 2: Conexion con la base de datos

Crear la base de datos en Supabase y conectarla a nuestro codigo

Código del Prompt:

He decidido usar Supabase como base de datos para alojar todo lo necesario para el programa.
Dime qué comando npm necesito para instalar el cliente de Supabase (@supabase/supabase-js).
Dame el código para configurar el cliente en src/supabase/client.js usando variables de entorno (VITE_SUPABASE_URL, etc.).
Diseño de la Base de Datos: Como Supabase usa SQL, dame el script SQL para crear la tabla notes en el editor de SQL de Supabase. La tabla debe tener:
id (uuid, primary key).
user_id (uuid, para saber de quién es la nota).
title (text).
content (text, para notas simples).
type (text: 'text', 'list', 'image', 'drawing').
is_pinned (boolean).
color (text).
list_items (jsonb, para guardar los items del checklist si es una lista).
created_at (timestamp).
Crea un Custom Hook useNotes que tenga funciones para leer las notas (SELECT), insertar (INSERT), actualizar (UPDATE) y borrar (DELETE).

Prompt 3: Aumentar las funcionalidades de la aplicación y conectarlo con la base de datos real

Pedimos que las notas se creen y la base de datos las guarda, ademas añadimos funcionalidades que tiene la aplicación original, como el modo oscuro o ver en listas.

Código del Prompt:

Pasemos con la siguiente fase del proyecto:
Vamos a conectar la lógica con la vista.
Modifica el componente CreateNoteInput. Haz que sea un formulario controlado (useState) y que al pulsar 'Cerrar' o hacer click fuera, llame a la función addNote de mi hook de Firebase.
Modifica NoteCard. Haz que los botones de 'Papelera' y 'Fijar' llamen a deleteNote y togglePin respectivamente.
Implementa la separación visual: En NoteList, filtra las notas para renderizar primero un bloque con las 'Fijadas' y luego otro con las 'Otras', igual que en el HTML original."
Añade las siguientes funcionalidades:
Para las notas:
Añade la capacidad de que las notas tengan recordatorios, etiquetas que creas y le das a las notas, la capacidad de ser archivadas, una zona donde se quedan las notas eliminadas antes de ser eliminadas del todo, la capacidad de poner colores a las notas e imagenes de fondo
Para la pagina:
Añade la capacidad de que se vean las notas en lista o en cuadricula, añade la capacidad de extender y plegar el menu principal, que el boton de buscar busque por recordatorios y etiquetas, añade un boton donde habilitar y deshabilitar el modo oscuro () y un boton de actualizar la pagina por si no se muestra nada.

Prompt 4: Usar Auth de Google

Conectamos nuestra app con el servicio de autentificación de Google, permitiendo al usuario logearse.

Código del Prompt:

Vamos a configurar la seguridad y el login.
Crea un componente Login que use el método supabase.auth.signInWithOAuth({ provider: 'google' }).
Por ultimo, crea un AuthProvider (Contexto) para detectar la sesión del usuario (onAuthStateChange).
IMPORTANTE (Seguridad): Explícame cómo activar las políticas RLS (Row Level Security) en mi tabla notes desde el panel de Supabase. Dame el código SQL de la política para que:
Los usuarios solo puedan hacer SELECT (ver) sus propias filas (auth.uid() = user_id).
Los usuarios solo puedan hacer INSERT/UPDATE/DELETE en sus propias filas.
Actualiza el hook useNotes para que incluya automáticamente el user_id al crear una nota nueva."

Archivo package.json mostrando el nombre y versión inicial.

Variables de Entorno (Seguridad)

Cliente de Supabase

Inicialización del cliente de Supabase usando las variables de entorno.

Habilitación del modo oscuro en tailwind.config.js y configuración de PostCSS.

Provider de Autenticación

Estado global para gestionar al usuario y el estado de carga.

Lógica de Login con Google

Uso de supabase.auth.signInWithOAuth para el inicio de sesión social.

Importación del AuthProvider y el componente Login en el punto de entrada.

Consumo del Contexto (Header)

Uso del hook useAuth en el Header para mostrar datos del usuario y la opción de salir.

Al crear una nota, se vincula automáticamente el ID del usuario autenticado.

Sincronización de Datos

Efecto que carga las notas solo cuando existe un usuario y las limpia al cerrar sesión.
